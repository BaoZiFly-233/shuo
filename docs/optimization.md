# 推理加速优化记录

音频 24.24s，模型 Qwen3-ASR-0.6B-ONNX-CPU（INT8 量化解码器）

## 优化项

### 1. ONNX 线程数调优

设置 `intra_op_num_threads` 控制每个算子内部并行度。

| 线程数 | 推理耗时 | RTF |
|--------|----------|-----|
| 0（默认） | 27.58s | 1.14x |
| 2 | 17.96s | 0.74x |
| 4 | 11.93s | 0.49x |
| **6** | **6.59s** | **0.27x** |
| 8 | 6.82s | 0.28x |

结论：`intra_op_num_threads=6` 最优，比默认快 4 倍。

### 2. 模型预热（Warm-up）

启动时先用一段短音频跑一次推理，让 ONNX Runtime 完成：

- 图编译和算子优化
- 内存池分配
- 线程池创建

后续推理跳过这些开销。实测预热后推理稳定，无首次运行的额外抖动。

### 3. 模型预加载

GUI 启动时在后台线程加载模型，用户看到"加载中..."提示。加载完成后立即就绪，录音结束直接推理，无需等待模型加载。

## 综合效果

| 阶段 | 耗时 |
|------|------|
| 冷启动（加载+推理） | 38.44s |
| 预热 + 线程调优后推理 | **6.59s** |

提升约 **5.8 倍**。
